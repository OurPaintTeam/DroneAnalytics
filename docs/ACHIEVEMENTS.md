# Техническое задание
## Из папок систем, компонент
 - Необходимо анализировать работу команд, их достижения. 
 - Создать картинки, для каждого достижения. 
 - Обновлять достижения команд должны авторизованные пользователи.
## Из папки процессов
### Базовые требования
 - Должен быть зафиксирован список достижений и набор метрик для фиксации новых достижений команд
 - Собираемая аналитика должна быть деперсонализированной (сбор и обработка персональных данных в сервисах аналитики запрещены)
 - Должны быть созданы изображения для каждого достижения
 - Должна быть создана и реализована концепция отображения достижений каждой команды
 - Просмотр текущих достижений каждой команды должен быть доступен через интернет в любое время
 - Только авторизованные пользователи должны иметь возможность влиять на алгоритмы аналитики и менять отображение достижений
### Расширенные функциональные требования
 - Результаты обработки аналитики и исходные данные должны собираться в базе данных (БД)
 - Должно быть реализовано регулярное создание резервных копий БД (не реже раза в неделю), должно быть проверено восстановление сервисов и данных из резервных копий.
 - Создание резервных копий и восстановление должно быть автоматизировано
### Примечания
 - в первой реализаций автоматизация может отсутствовать (т.е. ручной сбор и анализ данных)
# Реализация
 - План минимум: просто обновлять общедоступную таблицу
 - План максимум: Интегрировать в инфопанель

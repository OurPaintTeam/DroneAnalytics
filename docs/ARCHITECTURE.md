# Техническое задание
## Общие принципы
 - изоляция
 - контроль взаимодействия
 - минимализация доверенной вычислительной базы
 - использование брокера сообщений, мониторов безропасности
 - Контейнеризация, IaC
## Базовые принципы (интерпретация в контексте настоящей компоненты)
 - Взаимодействие между доменами безопасности происходит только через брокер сообщений
 - Выдача событий безопасности по запросу.
 - Передача телеметрик в сервис аналитики
 - Передача событий безопасностей в сервис аналитики
 - События должны записываться в журнал.
 - Сертификаты будут выдаваться и подписываться Регулятором. (То есть не сами подписываем TLS, а получаем его - Вроде)
## Основные функции
### Инфопанель (С16)
 - Обработка нового запроса (миссии)
 - Выполнение заданной миссии
 - Выполнение команд наземных управляющих систем
 - Передача телеметрии в наземные системы в процессе полёта
### Журнал событий (K11)
 - Инициализация хранилища для записи событий 
 - Запись нового события
 - Выгрузка записанных событий
### Аналитика (полученная потом и кровью)
 - метаданные по командам
 - по воздуху и в системе систем. Будет какой то серч, нужно будет отрабатывать какие соб. безоп. и т.д. Сколько каких систем, заказов и
потом эту инфу было бы не плохо визуализировать на инфопанели 
# Архитектура
## Фронтенд
Фронтенд выступает в этом проекте в своей, скажем так, инженерной аберации. Инфопанель выступает не внешним продуктом для массового 
потребителя, а продуктом для внутреннего пользования. Фронтенд должен выступать в качестве классического сайта в том случае, если важна 
вседоступность оного. Мы же, в связи с характером продукта, можем использовать классический десктоп. Это позволит ускорить разработку, а 
удар по производительности и доступности не принесет нам существенного вреда. В качестве целевой технологии предлагаю использовать PyQT,
ведь он удовлетворяет всем потребностям.  
Функции приложения не замысловатые и понятные, упоминания и уточнения не стоят.
## Сервис аналитики
Целью сервиса является сбор, анализ, агрегация и выдача данных с последующей визуализацией на инфопанели. Целевая технология - FastAPI.
Сервис лежит в локальной сети, имя доступ только к бэкенду и бд, для нас это означает, что все полученные запросы уже проверены и им
можно доверять, авторизация не требуется.
## База данных - Click House

# Техническое задание
## Базовые требования
 - Формулировка цели и предположений безопасности
 - авто тесты (GitHub Actions)
 - Код доверенных доменов безопасности должен быть покрыт тестами не менее чем на 60%
 - Код доверенных доменов безопасности, повышающих целостность данных, должен быть покрыт тестами не менее чем на 70%,
*включая зависимости*
 - Все зависимости должны быть проанализированы на предмет известных уязвимостей.
В доверенных доменах безопасности запрещается использовать сторонние пакеты с известными уязвимостями.
(План максимум - у нас не особо много зависимостей)
 -  Код остальных (недоверенных) доменов безопасности должен быть протестирован не менее чем на 30% 
(без учёта зависимостей).
